Hexo3 + Github 搭建个人博客

发表于 2017-09-29 | 阅读次数:
字数统计: 2,330 | 阅读时长 ≈ 9

前言

Hexo是一款基于Node.js的静态博客框架,依赖少易于安装使用，可以方便的生成静态网页托管在GitHub上，非常方便，遗憾的是，目前网上，用hexo搭建个人博客大多基于hexo2， 我结合自己搭建博客的实践与这两天踩的坑，总结一份教程，后来之人，能少踩一些坑，吾之愿也。

搭建步骤：

  • 配置Hexo安装环境
  • 安装Hexo
  • 配置Github与推送网站
  • 订制个性化域名
  • 一些基本配置
  • 尾声

配置Hexo安装环境

安装Node.js

安装Node.js会包含设置环境变量及npm的安装，安装后，检测Node.js是否安装成功，在命令行中输入： node -v ，检测npm是否安装成功，在命令行中输入：npm -v

安装Git

安装git后，要注册一个Github账号，然后打开Git Bash，将你的Git与GitHub帐号绑定，执行命令：

  • git config –global user.name “你的GitHub用户名”

  • git config –global user.email “你的GitHub注册邮箱”

如果想要了解Git的细节，参看廖雪峰老师的Git教程

安装hexo

换源

由于后续安装我们需要用用Node 的模块管理器 npm ，而Node 的官方模块仓库网速太慢，容易卡断，现在我们需要通过换源解决这个问题，有两种方式：

  1. 命令行指定（临时切换到阿里的源）：
    npm install –registry https://registry.npm.taobao.org info underscore + 包名

  2. 模块仓库切换到阿里的源，永久换源（建议用这种）：
    npm config set registry https://registry.npm.taobao.org/

确认是否切换成功，执行命令： npm config get registry

安装hexo

进入命令行，执行命令： npm install hexo-cli -g

查看hexo是否安装成功，执行命令：hexo version

然后依次按照下面步骤执行：

  • 创建文件夹myblog（名字随意）
  • 命令行cd 到myblog 里面
  • 初始化我们的博客,执行命令：hexo init myblog(名字随意,我们的站点文件将在其中生成）

若是出现如下提示，则表示初始化成功（下面两个warn 可以忽略，若是初始化不成功，请检查hexo是否安装好，是否换源）

  • npm install

等等，别急，虽然我们已经完成了最容易掉到坑里的部分，但是，后面还有一些小小的挑战等着我们，现在让我们扫扫hexo常用命令冷静一下：

hexo常用命令

npm install hexo -g #安装Hexo

npm update hexo -g #升级

hexo init #初始化博客（hexo init 若后接有文件名则新建文件，并初始化，否则对当前文件初始化）

命令简写

hexo n “我的博客” == hexo new “我的博客” #新建文章

hexo g == hexo generate #生成

hexo s == hexo server #启动服务预览

hexo d == hexo deploy #部署

hexo server #Hexo会监视文件变动并自动更新，无须重启服务器

hexo server -s #静态模式

hexo server -p 5000 #更改端口

hexo clean #清除缓存，若是网页正常情况下可以忽略这条命令

点击GitHub中的New repository创建新仓库，仓库名应该为：用户名 .github.io ,这是固定写法，可以观摩下图：

订制个性化域名

什么啊？我们的网址没有我们的标志，这不科学！

别急，这根本不叫事。

现在可以去阿里云或腾讯云买个域名，然后进入管理控制台的域名列表，找到你的个性化域名，进入解析，订制我们的个性化域名，此处用的是阿里云：

在上图三个箭头处，依次添加三条解析记录，如下：

A &npsp&npsp @ &npsp&npsp 192.30.252.153

A &npsp&npsp @ &npsp&npsp 192.30.252.154

CNAME &npsp&npsp www &npsp&npsp yourname.github.io

其中192.30.252.153\154是GitHub的地址,固定写法

然后，进入本地博客文件夹 ，进入myblog/source目录下，创建一个txt文件，输入你的域名，对，只要写进你自己买的的域名即可（带不带www都行，建议不带）：

如： helloqiang.top , 将此文件命名：CNAME（保存时去掉txt格式符）

现在需要登录GitHub，进入之前创建的仓库，点击settings，设置Custom domain，输入你的域名：

然后，打开myblog下的站点配置文件_config.yml，在url后添加你的博客地址，可以参考我的：

博客文章编辑

我们使用markdown，当然别的编辑器也可以，只不过markdown就是专门为那些经常码字或者进行文字排版的、对码字手速和排版顺畅度有要求的人群设计的，他们希望用键盘把文字内容啪啪啪地打出来后就已经排版好了。这些人包括经常需要写文档的码农、博客写手、网站小编、出版业人士等等，总之说的就是我们。

markdown的语法是一个小门槛，但是很简单就可以跨越，具体语法参看Markdown 语法说明(简体中文版)几分钟就可以入门哦。Markdown编辑器也有几种，这里推荐MarkPad 和The Markdown Editor for Windows

图床

图床，当博文中有图片时，若是少量图片，可以直接把图片存放在source文件夹中，但这显然不够的，因为图片多了之后会占据大量的存储的空间，加载的时候相对缓慢 ，这时考虑把博文里的图片上传到某一网站，然后获得外部链接，在文章中使用Markdown语法， 完成图片的插入，这种网站就被成为图床。国内比较好的图床有两个：新浪微博和 七牛云 ，注册一下，然后把图片传上去就好了，非常简单，他们的使用方法可以参看其他文章。
